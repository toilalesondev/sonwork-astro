An AI-native holding company operating system: one master brain that understands Son, builds products, and graduates mature products into their own teams, brains, systems, and companies.

The first product is the operating system that builds the next products.

Core Idea

1. Understand Son

The OS should know Son's principles, taste, goals, projects, context, decisions, network, and lessons so agents can build in his direction without constant re-explaining.

2. Build Continuously

The OS constantly saves knowledge, runs workflows, digests sources, builds prototypes, improves playbooks, and turns repeated work into automation.

3. Become A Holding

When a product matures, it gets its own brain, team, metrics, agents, and operating cadence. The parent holding allocates capital and governs the portfolio.

Parent-Child Structure

Son -> Holding Company OS -> Holding brain: thesis, principles, capital allocation, governance -> Shared services: research, build, content, sales ops, finance, evals -> Agent fleet: chief-of-staff, research, builder, memory, portfolio, eval -> Venture/Product OS -> Product brain: customers, roadmap, support, metrics, decisions -> Dedicated agents and humans when mature
Son = taste + final decisions Holding OS = capital allocation gbrain = memory Agents = workforce Skills = playbooks Cron = recurring operations

Venture Lifecycle

Do not create a separate company too early. Start inside the holding brain, then graduate only when traction and operational complexity justify it.

Stage 0IdeaCapture without commitment.
Stage 1ExperimentHypothesis, deadline, success metric.
Stage 2ProjectRoadmap, build loop, research, prototype.
Stage 3ProductUsers, support, pricing, metrics.
Stage 4VentureP&L, budget, dedicated agents, growth.
Stage 5CompanyHuman lead, own OS, reports to holding.

Weekly Cadence

Daily

Context Triage

What came in? What matters? What needs Son? What can agents do? What should be saved?

Weekly

Company Learning Loop

What did we learn, ship, break, automate, kill, or graduate? What decisions does Son need to make?

Monthly

Portfolio Review

Which bets deserve more capital? Which should stop? What product or thesis is emerging?

Quarterly

Strategy Reset

Pick top 1-3 bets, adjust capital allocation, decide hires, and update holding thesis.

Decision Rules

Graduate

Move an idea forward when it has user pain, a clear next test, repeated workflows, or traction.

Kill

Stop when there is no pull, no founder energy, unclear revenue path, or the learning loop is weak.

Spin Out

Create a venture/company only when it has customers, metrics, dedicated workflows, and needs autonomy.
