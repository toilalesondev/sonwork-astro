The shape — Model B

One brain, folders not walls

We follow Garry Tan's company-brain Model B: a single shared source — gbrain-shared — divided by folder convention, not by hard isolation. Every agent reads everything; write authority is split by namespace. No OAuth walls between Robin and Karpathy — just discipline.

son/        # Sir's personal — people, knowledge, ideas, notes
robin/      # Robin's workspace — ops, research, reports, inbox
karpathy/   # Karpathy's workspace — debugging, patterns, lessons
projects/   # canonical project pages (Perfeat, Sonwork, CorpRep)
fleet/      # shared ops — policies, incidents, x-digests
team/       # team decisions, architecture docs
The one rule

Before creating any page → read resolver.md. It's the filing decision tree. projects/ is the single source of truth for project pages — working notes about a project go in robin/ or karpathy/, never duplicated into the canonical page. Everything links; zero orphans.

Tenancy in practice

The sources

Lesson 06 explained isolation in theory. Here's how it's actually carved up. Two kinds of source live in our DB:

Human brain — shared, federated

The gbrain-shared / default source is where the agents think. Folder-namespaced, federated, read by everyone. ~1,500 pages of Son's actual knowledge.

Code mirrors — walled off

Every git repo gets its own code-* source, pinned by a .gbrain-source dotfile in the repo root. A coding agent working in perfeat-mobile literally cannot see Son's personal pages — enforced at SQL on every query. This is why the coding agents stay clean and the personal brain stays private.

The cog-* family are code-graph indexes (callers, callees, definitions) kept separate from prose so structural queries don't pollute semantic ones. The code-* hashes are immutable citation keys — the repo's identity, not its name.

The daily loop

How we drive it

Five verbs cover almost everything. The agents reach for the brain before the open web on any entity, person, or project lookup — that's the brain-first rule.

gbrain search "X"          # semantic + keyword, RRF-fused
gbrain get  son/people/x    # read a page by slug
gbrain put  son/notes/idea  # write-through: DB + disk + git, atomic
gbrain query "question"     # LLM answer, cited from chunks
gbrain think "hard one"     # multi-page synthesis, conflict + gap analysis
Write-through is the trick

gbrain put writes the database, the markdown file on disk, and the git commit simultaneously. The brain mirrors git 1:1. So a page Son edits on his Mac and a page Robin writes from the VPS converge to the same truth — no sync step to forget, no drift.

Then the dream cycle (Lesson 05) runs continuously via gbrain autopilot: it lints the new files, walks backlinks so nothing's orphaned, syncs git → DB, synthesizes transcripts into structure, extracts facts, re-embeds whatever went stale, and grades the beliefs. Nobody triggers it. Son wakes up and the brain is tidier than he left it.

Why this shape

What it buys us

For a one-person holding company

Son runs an AI holding company solo with two agents. The brain is the shared nervous system: every decision, person, project, and lesson is one put away from being permanent and searchable by both agents forever. Model B keeps it frictionless; isolated sources keep the code clean; the dream cycle keeps it from rotting. That's the whole bet — institutional memory without a team to hold it.

Remember
  • One shared brain (gbrain-shared), folder-namespaced — Model B, not hard isolation.
  • Code repos get walled-off code-* sources — agents can't leak personal pages into a repo's brain.
  • Write-through means DB + disk + git stay 1:1 — edits from Mac and VPS converge.
  • Brain-first — check gbrain before the open web on any lookup.
  • Autopilot runs the dream cycle continuously — the brain maintains itself.
