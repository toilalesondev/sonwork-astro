Drive Your Brain

OperatorMode

Four commands. By the end you can search, read, ask, and write to your own knowledge base from the terminal — no agent required.

The one thing this teaches

gbrain is just a CLI over your 1,385 markdown pages. Four verbs run 90% of daily use: search, get, query, put.

The mental model

Your brain is a folder of markdown files at /srv/gbrain-shared, mirrored into a database with embeddings. The CLI reads and writes both at once. Every page has a slug — its address, like son/projects/weber — and lives under a folder convention you already use: son/, robin/, projects/, fleet/.

Gotcha I hit live, teaching this

There is no --source flag. It throws Unknown command. You scope which brain you're hitting with an environment variable: GBRAIN_SOURCE=gbrain-shared. Your main brain is gbrain-shared (1,385 pages). Put it in front of any command.

The four verbs

VerbWhat it doesWhen
searchFast keyword match (tsvector). Returns scored slugs + snippets."Where's the page about X?"
getPrints a full page by its slug."Show me that page."
queryHybrid search + LLM answer over the brain. (ask is an alias.)"What do I know about X?"
putWrites/updates a page → DB + disk + git, at once."Save this."

01 — search

Find the page. Verified against your live brain just now:

# keyword search, scoped to your main brain
GBRAIN_SOURCE=gbrain-shared gbrain search "Weber"

# real output:
[0.8356] son/projects/weber -- # Weber — Brand & Landing Page Concept
[0.8224] son/projects/weber-target-market -- # Weber — Target Market...
[0.8300] son/projects/weber-niche-decision -- # Weber — Niche Decision...

The number is the relevance score. The slug after it is the page address — copy it for get.

02 — get

Read the page.

# feed a slug from search straight into get
GBRAIN_SOURCE=gbrain-shared gbrain get son/projects/weber

Prints the full markdown — frontmatter, body, wikilinks. This is the file on disk, verbatim.

03 — query

search finds pages; query reads them and answers you. It pulls the relevant pages and synthesizes.

GBRAIN_SOURCE=gbrain-shared gbrain query "what is the Weber project"

# --no-expand = skip query-expansion, faster & cheaper for simple asks
GBRAIN_SOURCE=gbrain-shared gbrain query "what is Weber" --no-expand

Rule of thumb: search when you know roughly what you want and need the address. query when you want an answer, not a list.

04 — put

The one that changes things. put is write-through: it updates the database, the markdown file on disk, and git — in one shot. The dream cycle (Lesson 3) then embeds + backlinks it automatically.

# pipe markdown in from a file
GBRAIN_SOURCE=gbrain-shared gbrain put son/notes/my-first-page < note.md
Slug = filing decision

The slug's folder prefix decides where it lives. son/ = your personal stuff, robin/ = my workspace, projects/ = canonical project pages. Before inventing a new path, the house rule is: read resolver.md first. Wrong prefix = orphaned page nobody finds.

Your turn — do this now (2 min)
  1. Run GBRAIN_SOURCE=gbrain-shared gbrain search "Perfeat"
  2. Pick a slug from the output, run gbrain get <slug> on it.
  3. Run GBRAIN_SOURCE=gbrain-shared gbrain query "what's the status of Perfeat" --no-expand
  4. Notice the difference between the list (search) and the answer (query).

Don't do put yet — we'll do a real write together in a follow-up so you don't mis-file. Reading is safe; writing needs the slug discipline from Lesson 3.

Make it permanent

Tired of typing GBRAIN_SOURCE=gbrain-shared every time? Two options:

# option A: export once per shell session
export GBRAIN_SOURCE=gbrain-shared

# option B: a .gbrain-source dotfile pins it for the working dir
echo "gbrain-shared" > ~/.gbrain-source
