# triage — issues through a role state machine

Move tracker issues through category + state roles. **Every AI-posted comment/issue during triage
must start with:** `> *This was generated by AI during triage.*` (Role→label mapping from
AGENTS.md / `setup-matt-pocock-skills`.)

## Trigger
- "Triage", "what needs my attention", "move #42 to ready-for-agent", "what's ready for agents",
  "create an issue".

## Roles
- **Category (one):** `bug` · `enhancement`.
- **State (one):** `needs-triage` · `needs-info` · `ready-for-agent` · `ready-for-human` · `wontfix`.
- Every triaged issue carries exactly one category + one state. Conflicting states → flag + ask.

## Procedure
- **State machine:** unlabeled → `needs-triage` → (`needs-info` | `ready-for-agent` |
  `ready-for-human` | `wontfix`). `needs-info` returns to `needs-triage` once the reporter replies.
  The maintainer can override anytime — flag unusual transitions and ask before proceeding.
- **Show what needs attention:** present three buckets, oldest first — (1) Unlabeled, (2)
  `needs-triage`, (3) `needs-info` with reporter activity since last triage. Counts + one-line
  summary per issue; let the maintainer pick.
- **Act on natural-language requests** ("show what needs attention", "look at #42", "move #42 to
  ready-for-agent", "what's ready for agents").
- **`ready-for-agent`** means fully specified — durable agent brief, acceptance criteria, no open
  questions. Don't mark it ready if a human still needs to decide something.

## Operating Rules — MANDATORY
Obey `.helmsman/harness/operating-rules.md`. Prefix every triage post with the AI disclaimer.
Exactly one category + one state per issue. Flag + ask on conflicts or unusual transitions —
the maintainer owns the call.

## Evidence (what "done" must show)
The issues moved + their new roles (category+state), the AI-disclaimer present on each post, and
any conflicts flagged for the maintainer.
